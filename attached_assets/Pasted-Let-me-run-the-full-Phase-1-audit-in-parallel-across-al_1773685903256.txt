How Pause is Persisted
POST /api/student/attempts/[id]/pause — validates Test.allowPause = true and status = IN_PROGRESS, then in a single DB transaction: creates AttemptPause { pausedAt: now } and sets Attempt.status = PAUSED.
POST /api/student/attempts/[id]/resume — finds the latest AttemptPause with resumedAt = null, sets its resumedAt = now, and sets Attempt.status = IN_PROGRESS.
On submit — any open pause event is force-closed with resumedAt = now so time accounting is always complete.
GET /api/student/attempts/[id] returns totalPausedMs computed server-side from all pause event durations.
How Timer Metadata is Exposed
timerMode in the test brief tells the FE exactly which timers to render:

"TOTAL" — only totalDurationSec matters
"SECTION" — render total timer + per-section durationSec
"SUBSECTION" — render total + section + subsection durationSec
Each section and subsection carries its own durationSec in the payload. The FE never needs to infer durations.
