H. HEAT MAP SYSTEM
==================================================

Build 3 rule-based heat maps.

----------------------------------------
H1. Accuracy Heat Map
----------------------------------------

Purpose:
Show strong and weak subject/topic/subtopic areas.

Per bucket:
- bucket_correct
- bucket_wrong
- bucket_unattempted
- bucket_attempted = bucket_correct + bucket_wrong

Formula:
bucket_accuracy_pct = (bucket_correct / max(bucket_attempted, 1)) * 100

Color logic:
- Green: accuracy >= 75
- Yellow: 50–74.99
- Orange: 30–49.99
- Red: < 30
- Gray: no useful attempt data / fully unattempted

Also provide a summary table with columns like:
- Subject/Topic
- Questions
- Attempted
- Correct
- Wrong
- Unattempted
- Accuracy %
- Status
- What it means

----------------------------------------
H2. Time Efficiency Heat Map
----------------------------------------

Purpose:
Show where learner is faster/slower than average.

Per attempted question:
- learner_q_time_ms
- avg_q_time_ms from first-attempt learners for that question

Formula:
time_ratio = learner_q_time_ms / max(avg_q_time_ms, 1)

Interpretation:
- < 0.85 = faster than average
- 0.85 to 1.15 = near average
- > 1.15 to 1.5 = slower than average
- > 1.5 = much slower than average

Also provide a student-friendly summary table with columns like:
- Topic / Question count
- Your avg time
- Avg student time
- Time ratio
- Status
- Suggested meaning

----------------------------------------
H3. Risk / Mistake Heat Map
----------------------------------------

Purpose:
Show the type of mistakes and time-behavior pattern.

Per question classify into one state:

1. Correct & Efficient
   - correct
   - time_ratio <= 1.15

2. Correct but Time-Heavy
   - correct
   - time_ratio > 1.15

3. Wrong & Rushed
   - wrong
   - time_ratio < 0.75

4. Wrong & Time-Heavy
   - wrong
   - time_ratio > 1.15

5. Unattempted
   - unanswered

Also provide a clean summary table such as:
- Question behavior type
- Count
- Meaning
- What to improve

This table is important for non-technical students.

==================================================
I. FOCUS AREA SCORING
==================================================

For each topic/subtopic, compute a Focus Score out of 100.

Inputs:

1. Accuracy risk
accuracy_risk = 100 - topic_accuracy_pct

2. Unattempted risk
topic_unattempted_rate_pct = (topic_unattempted / max(topic_total_questions, 1)) * 100

3. Time inefficiency risk
Based on topic_time_ratio:
- ratio <= 1.0 → 0 risk
- 1.0 < ratio <= 1.2 → 25 risk
- 1.2 < ratio <= 1.5 → 60 risk
- ratio > 1.5 → 100 risk

4. Wrong density risk
wrong_density_risk = (topic_wrong / max(topic_total_questions, 1)) * 100

Weighted formula:
focus_score =
  (0.45 * accuracy_risk) +
  (0.20 * topic_unattempted_rate_pct) +
  (0.20 * time_inefficiency_risk) +
  (0.15 * wrong_density_risk)

Priority labels:
- High Priority: score >= 70
- Moderate Priority: 45–69
- Low Priority: 25–44
- Strong Area: < 25

Also provide a summary table:
- Topic
- Accuracy %
- Unattempted %
- Time status
- Focus score
- Priority
- Action suggestion

J. SAPHALA MENTOR SUGGESTIONS (RULE-BASED ONLY)
==================================================

Do NOT use AI.

Generate suggestions only from explicit rule logic.

----------------------------------------
J1. Test-Level Suggestion Rules
----------------------------------------

Rule T1 — High speed, low accuracy
If:
- accuracy_pct < 55
- total_exam_time_ms < 0.85 * cohort_avg_total_time_ms

Suggestion:
You are solving faster than most learners, but your accuracy is low. Slow down slightly and read questions more carefully.

Rule T2 — Slow speed, low accuracy
If:
- accuracy_pct < 55
- total_exam_time_ms > 1.15 * cohort_avg_total_time_ms

Suggestion:
You are spending more time than average, but accuracy is still low. Focus on concept clarity and better question selection.

Rule T3 — Good accuracy, low attempts
If:
- accuracy_pct >= 70
- attempt_rate_pct < 75

Suggestion:
Your accuracy is good, but you are leaving too many questions unattempted. Work on speed and confidence.

Rule T4 — Good attempts, poor accuracy
If:
- attempt_rate_pct >= 85
- accuracy_pct < 55

Suggestion:
Your attempt rate is high, but too many answers are incorrect. Focus on controlled attempts instead of aggressive attempting.

Rule T5 — Strong overall performance
If:
- accuracy_pct >= 75
- attempt_rate_pct >= 80
- rank is strong / top band

Suggestion:
This is a strong performance. Maintain this speed-accuracy balance and focus on reducing small mistakes.

----------------------------------------
J2. Topic-Level Suggestion Rules
----------------------------------------

Rule P1 — Weak accuracy
If topic_accuracy_pct < 50
Suggestion:

Rule P2 — High unattempted
If topic_unattempted_rate_pct > 40
Suggestion:
You left many {topic} questions unattempted. This suggests low confidence or slow recall. Revise basics and do timed drills.

Rule P3 — Slow but correct
If:
- topic_accuracy_pct >= 70
- topic_time_ratio > 1.2
Suggestion:
You are getting {topic} questions correct, but taking more time than average. Improve speed through repeated timed practice.

Rule P4 — Fast but wrong
If:
- topic_accuracy_pct < 50
- topic_time_ratio < 0.85
Suggestion:
You are solving {topic} questions quickly, but accuracy is weak. Slow down and focus on careful reading.

Rule P5 — Strong topic
If:
- topic_accuracy_pct >= 75
- topic_time_ratio <= 1.1
- topic_unattempted_rate_pct <= 20
Suggestion:
{topic} is one of your strong areas. Maintain this performance and use it to secure confident marks.
