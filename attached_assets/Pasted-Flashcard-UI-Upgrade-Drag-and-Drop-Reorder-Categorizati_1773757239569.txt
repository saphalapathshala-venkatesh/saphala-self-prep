Flashcard UI Upgrade: Drag-and-Drop Reorder + Categorization

You are working on Saphala Self Prep flashcard player UI.

This task is only for frontend interaction upgrade of flashcards.

Read first, patch second.
Do not redesign unrelated modules.

Goals

Implement the following UX improvements:

1. Reorder cards

Current arrow-based reorder UI is not user-friendly.

Replace it with:

mouse drag-and-drop on laptop

touch drag-and-drop on mobile

Behavior:

student can drag items to reorder them

on submit:

each item position is evaluated

correct items shown in green

wrong items shown in red

if all correct:

show Flip Card button only

if at least one wrong:

show Try Again and Flip Card

Explanation should appear only after flipping.

2. Categorization cards

Implement true categorization interaction.

UI requirements:

categories must be shown as large visible boxes

items must be shown as small card/button-like draggable chips

student can drag an item into a category box

already-categorized items must be movable again into another category any number of times before submit

Evaluation:

on submit, inside each category box:

correct items shown in green

wrong items shown in red

Flow:

if all correct → Flip Card only

if at least one wrong → Try Again + Flip Card

Explanation appears only after flip.

Interaction requirements

Use a drag-and-drop approach that works for:

desktop mouse dragging

mobile touch dragging

Keep it smooth and simple.
Do not make the UI visually cluttered.

If an existing DnD library is already present and suitable, reuse it.
If not, add the smallest reliable solution.
