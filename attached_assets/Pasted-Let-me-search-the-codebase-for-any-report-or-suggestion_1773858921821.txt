Test Reports & Suggestions — Current State
Short answer: Neither feature exists yet. The data infrastructure is partially in place, but no student-facing report page and no suggestion/feedback system has been built. Here is exactly what is and isn't there:

What Currently Exists
1. Score Storage (backend only)

When a student submits a test, the system computes and saves these fields on the Attempt record:

Field	What it stores
scorePct	Percentage score (0–100)
correctCount	Number of correct answers
wrongCount	Number of wrong answers
unansweredCount	Questions left unattempted
totalTimeUsedMs	Total time taken in milliseconds
submittedAt	Exact time of submission
attemptNumber	Which attempt (1st, 2nd, 3rd...)
Per-question detail is also saved in AttemptAnswer:

Field	What it stores
selectedOptionIds	What the student selected
isCorrect	Whether it was correct
isMarkedForReview	Was it flagged for review during the test
timeSpentMs	How long the student spent on this question
2. Admin Analytics Page (admin-only)

There is a reports section in the admin console under /admin/analytics with four report types — but these are admin-only dashboards, not student-facing:

Daily attempt counts and unique users
XP points earned per day
Revenue per day (by payment stream)
Category-wise average scores
3. rrGenerated flag

The Attempt model has a rrGenerated boolean (Report Ready Generated). It is stored but nothing generates it or reads it yet — it was added as a placeholder for a future report generation pipeline.
